# UC-001: Core Data Model

## Status: IMPLEMENTED

## Objective Alignment
Links to: OBJ-001 ([Build a Production-Quality Re-Ranking Library](../objectives/OBJ-001.md))

## Description
Define the foundational data types (Document, Query, Token, Score, RankedResult, FieldWeight) that all other modules depend on. These must be immutable Java records with full validation, enabling parallel development of tokenizer, scorer, and pipeline modules.

## Beneficiaries
- Library users: clean, type-safe API for representing documents and queries
- Agent developers: stable types to code against when implementing scorers and tokenizers
- Pipeline builders: composable result types for ranking workflows
