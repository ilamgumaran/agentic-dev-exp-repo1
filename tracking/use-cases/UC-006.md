# UC-006: LLM Re-Ranking

## Status: SPEC'D

## Objective Alignment
Links to: OBJ-001 ([Build a Production-Quality Re-Ranking Library](../objectives/OBJ-001.md))

## Description
Provide an optional final-stage re-ranker that uses an LLM to judge relevance of the top candidates and merges the LLM's ranking with the base ranking. This delivers the quality of cross-encoder/LLM judgment while staying robust (a single bad LLM call cannot fully override solid base signals) and zero-dependency (the LLM call is a user-supplied functional interface).

## Beneficiaries
- **RAG / search application builders**: highest-quality final ranking on top-N candidates
- **DIY & e-commerce shoppers**: LLM understands intent ("gift for mom") better than token/vector matching
- **Library users**: opt-in capability; bring their own LLM via `LlmClient` (OpenAI, Anthropic, local model)

## Decisions
- [DR-001: Zero external dependencies](../decisions/DR-001.md) — LlmClient is a functional interface; no SDK in core
- [DR-004: Separate FusionStrategy from Aggregator](../decisions/DR-004.md) — LlmReRanker is a query-aware FusionStrategy
- [DR-005: LLM re-ranking as merge of two rankings](../decisions/DR-005.md) — merge base + LLM via RRF for robustness

## Success Criteria
- [ ] LlmReRanker implements query-aware FusionStrategy
- [ ] Merges base ranking + LLM ranking via RRF
- [ ] Graceful degradation on malformed LLM output (falls back to base)
- [ ] Zero new dependencies in core
- [ ] Testable with a deterministic fake LlmClient (no network)
