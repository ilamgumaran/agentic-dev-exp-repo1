# UC-005: Hybrid Search Rank Fusion

## Status: SPEC'D

## Objective Alignment
Links to: OBJ-001 ([Build a Production-Quality Re-Ranking Library](../objectives/OBJ-001.md))

## Description
Enable combining multiple scorers with incomparable score scales — specifically lexical (BM25) and semantic (vector cosine) scorers — using industry-standard rank fusion. Without this, naive score addition lets the larger-magnitude signal dominate by scale rather than relevance (the "score normalization problem"). This is the single most important enhancement for real-world hybrid search.

## Beneficiaries
- **All customer segments**: hybrid search (lexical + semantic) is what makes "blue running shoes" match "azure jogging sneakers" while still ranking exact matches first
- **DIY home improvers & e-commerce shoppers**: benefit most — their conceptual queries need semantic matching fused with keyword matching
- **Library users**: get a zero-tuning robust default (RRF) plus tunable normalized fusion
- **Contractors & financial analysts**: can lean on RRF with a low rank constant or weighted normalized fusion to keep exact matches dominant
