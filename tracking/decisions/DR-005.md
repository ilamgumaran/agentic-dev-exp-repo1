# DR-005: LLM Re-Ranking as a Merge of Two Rankings

## Status: ACCEPTED
## Date: 2026-05-28
## Decider: Ilam Gumaran

## Objective Alignment
Links to: OBJ-001 ([Build a Production-Quality Re-Ranking Library](../objectives/OBJ-001.md))
Impact on metrics: Enhancement Ease Score (composes with existing fusion), no impact on core dependency count

## Context
Adding optional LLM-based re-ranking. Two design questions:
1. How to keep `reranker-core` zero-dependency while calling an LLM (which needs network/SDK)?
2. How to use the LLM's judgment without letting a single bad/hallucinated response wreck results?

## Options Considered

### Option A: Built-in HTTP client calling a specific LLM API
- Pros: Works out of the box
- Cons: Breaks zero-dependency constraint; locks users to one provider; network code in core

### Option B: LLM fully replaces the ranking
- Pros: Simple
- Cons: A single bad LLM call destroys results; no robustness; expensive on full corpus

### Option C: User-supplied LlmClient functional interface + merge LLM ranking with base ranking via RRF
- Pros: Zero-dependency (user wires their LLM); robust (merge, not override); top-N only (cheap); testable with a fake client
- Cons: User must implement LlmClient; two rankings to reason about

## Decision
Option C. `LlmClient` is a `@FunctionalInterface` the user implements. `LlmReRanker` is a query-aware `FusionStrategy` that:
1. Gets a base ranking from a wrapped FusionStrategy (result 1)
2. Asks the LLM to re-rank the top-N candidates (result 2)
3. Merges the two rankings via RRF

This "merge two results" approach gives the quality lift of LLM judgment with robustness against LLM errors, while keeping core zero-dependency.

## Consequences
- Core stays zero-dependency; LLM SDK/HTTP lives in user code
- Robust to LLM hallucination/outage (graceful degradation to base ranking)
- Required adding `Query` to the `FusionStrategy.fuse()` signature (spec 005 updated) so query-aware strategies can build prompts
- LLM called at most once per fuse() (batched), controlling cost
- Users bring any provider (OpenAI, Anthropic, local)
