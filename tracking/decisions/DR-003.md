# DR-003: Multi-Agent Generic Configuration

## Status: ACCEPTED
## Date: 2026-05-27
## Decider: Ilam Gumaran

## Objective Alignment
Links to: OBJ-002 ([Validate Agentic Development Workflow](../objectives/OBJ-002.md))
Impact on metrics: Agent Success Rate, Enhancement Ease Score

## Context
This repo is an experiment in agentic development. To validate the workflow across agents, the repo must be compatible with Claude Code, Codex, Copilot, Gemini, and Q Developer — not locked to one agent.

## Options Considered

### Option A: Claude Code Only
- Pros: Simple, one config file
- Cons: Can't validate multi-agent workflows, vendor lock-in

### Option B: Generic + Agent-Specific
- Pros: Any agent works, shared rules prevent duplication, agent-specific configs for platform features
- Cons: More files to maintain, risk of divergence between generic and specific

## Decision
Option B: Generic `.agent-config/AGENT_RULES.md` as single source of truth, with agent-specific files (CLAUDE.md, GEMINI.md, copilot-instructions.md) referencing it.

## Consequences
- Any new agent platform can be added by creating one small config file
- Shared rules updated in one place
- Agent-specific configs must stay thin (platform features only, not duplicated rules)
